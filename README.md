# 👻 Ghost Entity Logic (Java OOP)

This project simulates the core logic and state management for ghost-like characters in a game environment. It focuses on clean code principles and the practical application of **Object-Oriented Programming**.

## 🚀 Technical Features
- **Constructor Overloading**: Implements multiple ways to initialize entities (Default, Partial, and Full parameters) to handle different spawning scenarios.
- **State Management**: Robust logic for life tracking (`Morir()`), speed increments (`Acelerar()`), and visual status updates.
- **Encapsulation**: Strict use of private fields with public Getters and Setters to maintain data integrity.
- **UTF-8 Support**: Configured to handle standard character encoding for consistent console output.

## 🛠️ Game Mechanics
- **Spawning System**: Visual confirmation when an entity enters the game world.
- **Dynamic Movement**: Real-time velocity updates with validation.
- **Life Cycle**: A complete system that tracks health points and triggers a "GAME OVER" state once lives reach zero.
